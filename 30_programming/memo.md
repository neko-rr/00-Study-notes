# 未整理
各ツールは何のために必要か（初学者向け）
TypeScript

目的：型注釈でコードの誤りを早期に検出する。共有パッケージの型を他のパッケージで安全に使える。

効果：関数の引数や戻り値のミスをコンパイル時に見つけられる。

zod

目的：ランタイムでデータ（APIレスポンスや外部入力）を検証するスキーマライブラリ。TypeScript の型と連携しやすい。

効果：フロントで受け取った API データが期待どおりかを実行時にチェックして、安全に扱える。

ESLint / Prettier

目的：コード品質とスタイルを自動で保つ。ESLint は静的解析、Prettier は整形。

効果：チームで一貫したコードスタイルになり、レビューが楽になる。

Pydantic（Python）

目的：バックエンドで受け取る/返すデータのバリデーションと型定義。

効果：FastAPI 等と相性が良く、API の入出力を厳密に扱える。
