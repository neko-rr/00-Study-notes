# 基礎
- $はコマンドの入力開始点に用意されている記号なので、書く必要はありません。
- コマンドは入力した後にEnterキーを押して実行
- コマンドラインには補完機能というものがあります。補完機能とは下の図のように、ファイル名やフォルダ名を入力するときにファイル名の入力途中にTabキーを押すことで、残りのファイル名を補完してくれる機能です。これを使うことにより、効率が良くなるだけでなく、先程のような存在しないファイル名によるエラーなどを防ぎやすくなります。
- フォルダをディレクトリと呼ぶ
  - カレントディレクトリ：自分が今いるディレクトリ
  - 1番親のディレクトリ：ルートディレクトリ。「/」で表されます。
  - ホームディレクトリ：ユーザーが作業をするときに起点にするディレクトリのこと
# ファイルの作成：touch
touch ファイル名（※拡張子含む）
# ファイルの中身を表示：cat
cat ファイル名
## 【エラー】ファイルの中身を表示：cat
存在しないファイルを指定するとエラーが出力されます。エラーが出た場合、実行したコマンドは無効
# ディレクトリを作成：mkdir
mkdir ディレクトリ名
# ディレクトリを移動する：cd
cd ディレクトリ名
- 1つ親のディレクトリに移動するときに「cd 1つ親のディレクトリ名」としても移動することができません。
  - 1つ親のディレクトリは、「..」で指定
  - 「cd ..」：1つ親のディレクトリに移動したい場合(cd + 半角空白 + ..）
- ディレクトリを指定せずに「cd」のみで実行すると、ホームディレクトリと呼ばれるディレクトリに移動する
  - ホームディレクトリは、下の図のように「~」で表されます
## 【エラー】ディレクトリを移動する：cd
存在しないディレクトリを指定するとエラーが出力されます。また、指定できるのはディレクトリ名だけでファイル名を指定することはできませんので注意してください。
# カレントディレクトリを確認：pwd
pwd
# ディレクトリの中身を表示：ls（エルエス）
ls
- lsコマンドで確認できるディレクトリとファイルは、基本的には1つ子の階層だけ
# 移動させる・変更する：mv
## ファイルを移動：mv
mv ファイル名 ディレクトリ名
## ディレクトリごと移動：mv
mv ディレクトリ名 移動先のディレクトリ名
## ファイル名を変更：mv
mv ファイル名 新しいファイル名
- ディレクトリ名も同じ
# コピー：cp
## ファイルのコピー：cp
cp コピーするファイル名 新しいファイル名
## ディレクトリのコピー：cp
cp -r コピーするディレクトリ名 新しいディレクトリ名
- 「-r」をつけずにディレクトリをコピーしようとすると、エラーが出力され、コピーは実行されません#
# 削除：rm
## ファイルの削除：rm
rm ファイル名
## ディレクトリの削除：rm
rm -r ディレクトリ
# コマンドの履歴
過去に実行したコマンドを上矢印キー「↑」で入力することができます。少し前に実行したコマンドを使いたいときには、とても便利
# 基本エラー
- エラー1: bash: pwd: command not found
  - ヒント: コマンドが正しく入力されているか確認してください。環境設定が間違っている可能性もあります。
- エラー2: mkdir: cannot create directory ‘project’: File exists
  - ヒント: project ディレクトリがすでに存在している可能性があります。ディレクトリ名を変更するか、ls コマンドで確認してください。
- エラー3: bash: cd: frontend: No such file or directory
  - ヒント: frontend ディレクトリが正しく作成されているか確認し、スペルミスがないかも確認してください。
- エラー4: bash: touch: cannot touch 'schema.sql': No such file or directory
  - ヒント: database ディレクトリが作成されているか確認し、ファイルのパスが正確か確認してください。
- エラー5: mkdir: cannot create directory 'build': File exists
  - ヒント: 同名のディレクトリがすでに存在している可能性があります。別の名前を使用するか、既存のディレクトリを確認してから進めてください。
- エラー6: rmdir: failed to remove 'components': Directory not empty
  - ヒント: ディレクトリ内にまだファイルが存在している可能性があります。すべてのファイルを削除してからディレクトリ削除を再試行してください。

# ripgrepで固定色検索
## 最強：全検索
```bash
rg -n "(#[0-9a-fA-F]{3,8}|rgba?\(|style=\{[^}]*\(color|background|border\))"
```
## 個別指定検索
1. hexカラー（3〜8桁）
```bash
rg -n "#[0-9a-fA-F]{3,8}"
```
2. rgb / rgba
```bash
rg -n "rgba?\("
```
3. JSX の style={} 内の color/background/border
```bash
rg -n "style=\{[^}]*\(color|background|border\)"
```
4. CSS の color/background/border に直書き
```bash
rg -n "(color|background|border)[^;]*#"
```
## 検索後
よくある固定色の例（grep対象）
種類	例
hex	#fff, #000000, #12abcf, #12345678
rgb	rgb(255, 0, 0)
rgba	rgba(255, 0, 0, 0.5)
JSX style	style={{ color: "#fff" }}
CSS	background: #f00;
