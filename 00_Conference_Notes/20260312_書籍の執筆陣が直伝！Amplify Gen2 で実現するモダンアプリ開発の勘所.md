# Amplify gen2を使うと何が嬉しいか	soccerSE
- バックエンドエンジニア
  - 4・7・10章の一部担当
- Amplify gen2
  - CI/CDの構築を一瞬でできる
  - 方法：GitHubと連携するだけ
  - プレビュー機能
    - プルリクエスト用の環境を作ってくれる
    - 一時的な画面なので、プルリクエストがマージされたら消える
      - 営業さんとかにも操作を試してもらえるのが良い
  - CDKベースでバックエンドを構築する手法がある
    - 定義してないけれど、デフォルトで作ってくれる
  - 注意点
    - DynamoDBのテーブル設計にこだわりたいなら、手動でテーブルを定義するのがおすすめ
      - アップデートされたGSIマルチキーサポートには対応していない
- ローカル開発における開発者体験が非常に良い
  - 開発者個人のバックエンドをやってくれる
```
npx ampx sandbox
```
# Cognito で認証画面をサクッと作る	fffk10
- シニアエンジニア・スクラムマスター
  - 5章を中心に7章・10章の一部担当
- Cognito
  - ユーザー認証サービス
    - ユーザー数に応じた従量課金
    - ベストプラクティスが自動で適応
- Managed Login
  - AWSが推奨するホスト型ログイン
  - プランディングエディターで色とかUIも変更可
  - 日本語化は、クエリに追加するだけ
  - 注意
    - ログイン時にリダイレクトが発生する
    - UIの自由度は、プランディングエディター程度に限定
- Hosted UI（レガシー）
  - 非推奨
- Amplify UI
  - AWS提供のUIコンポーネント集
  - リダイレクト不要。自分のアプリ内で確認可能
  - UIのカスタマイズが自由にできる
    - 実践では、この場合が多いだろう
    - ヘッドレスUIとしても使える！
      - 既存のデザインシステムをそのまま使える
      - 認証はAWS任せにできる
  - スクリーンショットの安定性（書籍採用理由・画面固定したかった）
# 一番頭のいい基盤モデルをBedrockで選びたい	Tomo
- 本、6章中心
- 本に書いていない事
  - モデル選定
- 現在、206モデル
- なぜ各社APIではなく、Bedrockを使うのか？
  - 最強の盾（セキュリティ）
    - サーバーの場所
      - 中国系でも安心して使える。。。
  - 学習利用ゼロ
  - 統一インターフェース
    - Converse APIで、全モデルを同じコードで「着せ替え」可能
- AWSの刺客
  - Amazon Novaファミリー
  - AWSでしか使えない、コストと機能のバランスプレイカー
- オススメ
  - 用途別
    - 最強のモデルは、存在しない
  - 最近モデルの比較モードが出たので、試してみよう（同じプロンプト）
- 決め方
  - 用途で絞る
    - コストｘ精度で天秤にかける
      - 実際に比較して試す
