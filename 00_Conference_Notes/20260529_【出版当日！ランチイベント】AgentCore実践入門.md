# Strands Agents入門（森田さん）
- Strands Agents
  - モデルxツールxプロンプトだけで、AIエージェントを動かすSDK
  - モデル非依存
  - Agent Loop
  - PythonとTypeScriptの二系統。基本機能は、ほぼ同等
- 書籍は、Pythonメインで書いている
- Pythonの方が、実験的機能やリリースの速さが早い
- どちらを選ぶ？
  - 組み込み先の環境と開発者のスキルセットで選ぶ
- AgentCoreで動かしたいなら、現在はPythonがオススメ
- 8行で、ツール付きエージェントが動く
- 構造がほぼ同じで、APIの形が揃っているので、TypeScriptもすぐに書ける
- どうして使うか？
  - 素のSDKで書くと、コード量多い
  - 手書きで必要な行数～60行
- 補助機能
  - セッション管理
  - フック Hooks
  - 会話管理
  - 構造化出力
- マルチエージェント構成
  - 4種類の組み合わせ方
- エコシステム
  - 本体だけでなく、
# Bedrock AgentCore入門（みのるん）
- 「問題児」を動かすインフラ選定の課題
- これまでは、クラウドの機能を自分で組み合わせるしかなかった
  - Bedrock AgentCoreが救世主で全部取り入れられた
    - 会社で、必要な機能が一通り盛り込まれた
- 利用例
  - 作ったエージェントをコンテナにしてデプロイするだけ
- エンプラの開発、デプロイして終わりじゃない
  - セキュアに運用・改善し続ける必要がある
    - これが、強い
- よくある課題
  - 部長専用のSharePointが見れちゃった
    - AgentCOreアイデンティティ機能
      - ツール関数をデコるだけ
    - 特定の操作をさせたくない
      - AgentCOreポリシー
    - リリースしたら終わり？監視したい
      - AgentCOreオブザーバビリティ
        - トレースを自動でいい感じに可視化！
        - PoC程度なら、これで良いじゃん
    - リリースしたけれど、ユーザーは満足しているの？
      - 選んだメトリクスで、エージェントの行動を自動で採点
      - 人間は、「これまずいかも？」というログを絞って改善に集中できる
    - 社内のエージェントやツールが増えすぎて、管理できない&野良乱立＆被り
      - AWSエージェントレジストリ
        - 利用者が探して、類似があるか探す
          - 感想：探す訳ないやん！
    - コード必須？
      - AgentCOreハーネスは、クリックで作れる
    - フロントエンド苦手問題
      - Reactを使用：Amplify Gen2
    - インフラ
      - インフラ全体をAWS CDKでコード管理すると、AIエージェントで管理しやすい
