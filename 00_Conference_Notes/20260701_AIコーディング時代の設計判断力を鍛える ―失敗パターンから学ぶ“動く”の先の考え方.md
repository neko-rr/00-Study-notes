# 内容
- ”動く”は、判断した事にならない
- 失敗パターンは昔からあったが、観測される頻度が桁違いに増えた
- 設計判断
  - 暗黙知
  - 言語化された教材も、明示的な育成プランも少ない
  - Coding Agent時代には、ベテランの背中が見える環境が少ない
  - Coding Agnetは、言わなければ判断軸を勝手に判断する
- 設計判断で使う8つの評価軸
  - 目的整合性
  - 制約整合性
    - 性能・セキュリティ・法令・制約
  - 実現可能性（技術・組織）
  - 品質影響
  - 時間効果（短期・長期）
  - リスク・不確実性
  - 整合性
  - 合意可能性
- 設計判断の失敗は「支配軸の取り違え」で発生する
- 評価軸が変われば、選ばれる手段も変わる
- 評価軸のトレードオフ
  - ATRを残す
  - AIも読める
- 設計判断失敗パターン
  - 評価軸の偏り記法
    - ↑過大評価　↓過小評価
    - 早合点
      - 「200が返った」を目的達成と混同する
    - 見切り発車
    - 牛刀を持って鶏を捌く
      - 要件規模に対して過剰な技術を選ぶ
    - 車輪の再発明
      - 標準ソリューションがあるのに独自実装する
    - 身の丈に合わない設計
      - 技術的には理想だが、運用が回らない
    - 羹に懲りて　を吹く
      - 組織スキルを過小評価して、優れた手段を「危ない」と退ける
    - 過去への忖度
      - 互換性の実利用を確認せず、最大限
    - 郷に従わぬ正論
      - 既存の規約を無視して、一般論で「正しい」書き方を持ち込む
    - 16パターンの失敗例を載せています
- 見抜く
- 失敗パターンを量産するskillを作ったので、設計判断失敗パターンを体験しよう
