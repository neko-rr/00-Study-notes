# 内容
- 感想：この前AWSの本書いて発表していた人（脆弱性の）
- Azureの用語集を書く時に苦労した事
  -AWSは、公式がPDFもアイコンも用意してくれている
  - 複数のサイトがあり、情報が統一されているのかと思いきや、実際は内容バラバラ。
    - アイコンとサイトのカテゴリー分類も異なる
    - サイトで含まれるサービスがバラバラ
    - 比較した結果、一番下のサイト（太字）がすぐに最新情報が更新されていたので、それを重視した（笑）
- 1章はAWSと一緒の事だが、多少変えている
  - Azure独特の体系とか
    - 費用体系とか
    - アカウントの配下にサブスクリプション、リソースグループという概念を持っています。
      - アカウント⇒ユーザー⇒サブスクリプション毎にアカウント制御 
  - ガイドラインが豊富
    - Gitに載っている
    - AWSのような5つの柱があるWell-アーチテクチャ
  - ある物は使用しましょう：アーキテクチャパターンとか
- 2章：インフラアーキテクチャ
  - Azureは、SQL Serverが充実
    - 大きく3種類のサービス
      - ユーザー側の管理範囲が違う
  - NoSQLデータベースの種類
    - AWSは個別サービス
    - Azureでも独自データベースがあるが、NoSQLのCosmoDBカスタマイズ
  - ストレージの種類は多い
- 3章：実行アーキテクチャ
  - Azureにはプランというものがあり、提供されるサービスが異なる
    - お金を出さないと使えるものに制限があるから、やっかい
    - お金出せば、結構できる
  - プランとサポートしている機能例
    - 実際に使用始める時は、まずプランを見て、何を使いたいか考える所から始める
    - AWSはサービスの可能な事が決まっているから、開始してからパズル
  - 典型的なサーバーレスの構成図
- 4章：データ連携基盤の構成説明
