# 「自分のハンドルを握る〜AI時代だからこそ求められるセルフマネジメントの技術」	小田中 育生 氏（いくおさん）
- 変化の中で自律的でいることは簡単ではない
- 心理的プレッシャー
  - 「追いつかなければ」という焦り「置いていかれる」恐怖（FOMO）
  - ⇒他律的な行動
    - 「ねばならない」が優先され、自分の行動が外部から規定される
- 変化が速い時代だからこそ、自分のハンドルを握り直そう
- セルフマネジメントの構造
  - キャリア・マネジメント
    - タスク・タイムマネジメント
    - アンガー・ストレスマネジメント
    - スキルマネジメント
      - モチベーション・マネジメント（土台）
- 外部環境が焦りを生む
  - 「やらねば」に追われる
  - 「やりたい」を取り戻す
    - それをやりたいと思っているのか
    - なぜそれをやりたいのか
    - 時間を忘れて没頭できる事は
- モチベーションはあっても、「時間がない」
  - AI時代のジェボンズのパラドックス
    - 技術は仕事を速くしたが、新たな仕事が出て来る
- 仕組で時間を取り戻す
  - 時間の使い方を設計する
    - WIP=1
    - タイムブロッキング
    - タイムボクシング
    - カレンダーのデフラグ
  - タスクを取捨選択する
    - タスクの細分化
    - 優先順位づけ
    - 他者への委譲
- 今こそYAGNI
  - 「必要だからやる」
- 時間とタスクがマネジメント出来ていても、感情がゆらいでしまう
  - 感情の出どころを探る
  - 感情の揺れを認め、リフレーミングする
    - 「解釈」が感情を生む
    - 解釈を変えれば、感情も変わる
- AI時代だからこそスキルマネジメントする
  - 何をAIに任せて、何を自分で学ぶか
  - 大切な姿勢：何でこれで動くんだっけ？と深堀
  - マイクロラーニング習慣を入れる
- Q&A　何をAIに任せていますか？
  - obsidianにナレッジ貯めているから、そこの整理とか
  - 自分の欠けている所を補うようにAIを使用
# 「チームパフォーマンスを高めるための2種類のセルフマネジメント」	岩瀬 義昌 氏（iwashiさん）
- チームマネジメントは、おまけでセルフマネジメント
- わざと、前の発表者の本を読む前に書いた
  - 結構、被っている
- セルフマネジメントは、チームにも効く
- シンプルに心身等の状態による
- n=1の実践例を挙げていきます（育休中）
  - 放送大学の学生
    - 耳は空いているけれど、手と目が塞ぐ事が多いので、人文学系の授業を取りまくっている
- 仕事
  - 時間とエネルギー
    - 「実際に働いた時間」と「働いたと感じる時間」は異なる
    - 自分は、どんなタスクでエネルギーを消費しやすいのか？
    - 自分の疲れやすい仕事やタスクを把握する
  - Getting Things Done
    - 仕掛りが増えると脳内費用が増える
- FOMO
  - 話者自身：朝起きてXを眺めると、海外のAIの情報が溢れている
    - ただ、実際には別に見てなくても何にも困らない
  - 完璧主義（人による）
    - 真面目に認知行動療法などに取り組むのも手
      - 放送大学の授業
      - CBT
        - 考え方（認知）と行動の癖を見直す
        - ABC（DE）理論
  - 自分の長期のゴールを書いてみる
    - 1年に1回程度、自分が長期的にやりたい事を言語化している
      - これによって、「やらない」、「優先度を下げる」を判断しやすくする
- 経験上上手くいかなかったもの
  - いわゆる心がけ
    - スマホを触る時間を減らそう
  - 締め切りのない緩い目標
    - 年末に本を読もう
  - Booleanがtrueにほぼならないやつ
    - やる気が出たらやる
    - 忙しくなくなったらやる
- 上手くいったもの
  - 行動じゃなくて環境を変える
  - 積読は、見えるところに（綺麗にレイアウト）
  - 作業机には必要な物だけ出す
    - SNSアプリをホーム画面から消す（消せないにしても通知を切る）

# 「自律的にセルフマネジメントできるチームのつくり方」	河合俊典 氏（ばんくしさん）
- 感想：本日2回目（笑）
- カジュアル面談で普通に人生相談とかやっています
- エムスリー会社説明
  - セルフマネジメント出来る事をかなり重視している
  - コンパクトな組織
    - 社員650名
    - エンジニア全体で120名、25チーム
      - 例：電子カルテシェア1位は10人くらいで作っている
  - フラットな関係性
    - ほとんどの事をチームが意思決定
    - エンジニアも含めCEO/CPO/CTOと直接議論
- 人を増やさない構造を体現し続けるのは、難しい
  - 重要なのが、チームの意思決定を強めるセルフマネジメント
- セルフマネジメントが出来ているチームは、心地良い
  - エンジニアリングの全てが可能になる
  - 外から何も言われない
  - 事業、チーム、個のエンジニアとして成長できる
- あらゆる外からの要望をコントロールできる能力がある
- 周囲がセルフマネジメント出来ていると、自分もつられて出来るようになる（しようとする）
- セルフマネジメントできるチームの要素
  - 意思決定
    - 事業を伸ばす、という事に終止できる
  - 挑戦
    - 飛躍的に拡大する未知のアイデアを発案し実行できる
  - 倫理
    - 収支に対して、市場に対して敏感である
- 資本主義
  - 企業活動は、相手もチーム
- セルフマネジメントの成長スパイラル
  - 意思決定
    - チームで決める
      - という事は、かくも難しい事である
    - 技術力に差があるのが当然で、一番技術力高い人がやればよいとなりがち
      - 人は増えても開発は良くならない
      - 組織が育たない
      - 意思決定の不文律、非対称性、制約
        - ROIを日常の意思決定に取り組む
  - 挑戦
    - ものづくりは停滞する
    - 人の成長は停滞する
    - 変化に対して市場は期待している
    - 挑戦の思考を定期的に実践する
    - 毎週1時間マネジメント会議で検討している
  - 倫理
    - 挑戦は必要、倫理は必須
    - 収支に気を遣う
    - 市場からの反応を考えられるか
    - 個人のROI、チームのROI
    - 費用対効果
      - 個人にとっては得だが、チームではマイナス
      - 会社によって異なる
# パネルディスカッション「ゲストが日常的に実践しているセルフマネジメント術」	モデレーター：ばんくしさん

    - それにならないように、気楽に生きるを意識している
      - 転機があり、プライベートに時間を使いたいため
    - カレンダーのデフラグ：1on1を寄せる
  - いくおさん：カレンダーのデフラグが好き
    - これが達成したら終わりを決めて、無限に時間を使わないようにしている
    - 44歳で体が衰えてきたので、毎日2万歩歩くようにしている
      - 歩かないと、30歳後半から仕事のパフォーマンスに影響でるように
