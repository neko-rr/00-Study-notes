# パネルディスカッション + Q&A
テーマ:私達は開発プロセスの将来をこう考えている
- 開発者側によってくる。Cursorからテストできたり。
  - それ以上にドメイン側のテスト要望が増えている。例は、建設業法に適応しているかとか。
  - トータルで品質上がってくる。
- E2Eの方でも出来る。開発する前からテストが出来ていて、それに合格するように作成していき徐々に合格率が上がっていく。
- 生成AIがあると、目や耳として動いてくれる。
- 課題:検証結果を担保する
  - 裏側のモデルが進化してる&複数モデル活用(両方で検証)
  - 客に見える側:AIアサーションを表示しています
  - ユーザー側と見えない側の2つあり。

- QAエンジニアが不要になりますか？
  - ⇒不要になる可能性はある。単純なスクショとかは要らなくなる。
  - エレメントオタクである事に価値が無くなってくる。
  - 但し、業界毎の業務毎のドメイン知識を活用したテストが設計できる人は残る。そして、自動ツールを使いこなす。

- コードを書くのが早くなってもテストが遅くて、ボトルネックになってはいけない。
  - ⇒将来は、両方とても早くなる。
