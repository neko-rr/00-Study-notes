# 12:02~12:20	『フルリモートのオンボーディングで早く戦力になるために必要なこと（仮）』	株式会社LayerX	穴井 宏幸さん
- 入社前の期待値調整
  - 「お手並み拝見」にしない配慮
    - 入社後1～2年は、配慮
  - 個人ごとのオンボーディングTODOページ
    - これを見て、どこまで進んでいるかのやり取りが可能
  - モノレポによる開発環境の統一
    - セットアップの高速化
      - 充実したドキュメントとツールにより、30分以内でローカル環境構築が完了
  - Timesチャンネルがない文化
    - オープンチャンネルでのコミュニケーションが推奨
      - 業務の話は公開チャンネル・雑談は専用チャンネル
    - 皆、見ていて、質問してもすぐに5，6人集まって来る
  - 新機能へのアサイン
    - 入社後、1ヶ月くらいで大きめの新機能へのアサイン
      - 新機能だと、既存との整合を考える箇所が少なくてやりやすく、自分の領域もできた
  - 開発チーム内で夕会の一言コーナー
    - 仕事に関係ない近況報告
    - 集まって1人、1～2分雑談：全員話すターンがある
- 何が良かったか？
  - モノレポ最高！
    - ドキュメントが古いがほぼ無かった
    - 困った時にチーム外からサポートが受けられる
  - モノレポに限らず、レバレッジが効く状態であれば良さそう
    - 開発等のコスト＜開発効率
  - Timesがない文化＝「聞いた方が早い」矯正ギブス
    - 聞いている内に調べ方も分かって来るので、自走も早くなる
  - 夕会の一言コーナー
    - 個人的には雑談したい派・したくない派の両方にちょうどいい感じがする
    - 気を使わない範囲の雑談
# 12:20~12:38	『「いつでも聞けるAI」と「毎日話せるメンター」とのオンボーディング』	ファインディ株式会社	前田 啓太さん
- 入社前の不安
  - フルリモートやっていける
  - 業務でがっつりAI活用した事ないけど、やっていける
- 最初の出社（2泊3日）のタイミング
  - 会社全体のオンボーディング
  - チームのエンジニア・PdM・デザイナーとひたすら自己紹介1on1
    - 皆、出社してくれる
  - GitHub Issueでオンボーディング期間でやることのチェックリストの確認
    - テンプレートが用意してあって、徐々に自分用にしていく
  - 開発環境もセット
- フルリモート期間
  - 毎朝15分のメンター1on1
    - 今日やることの確認
    - 「自分の理解、合っていますか？」のファクトチェック
  - 「いつでも聞けるAI」の活用
    - 各種AIツールが使用可能
    - MTGの録画・録音を行う
      - LLMに読み込ます
- オンボーディング1カ月目
  - 初月の目標設定
    - 開発の流れ・コミュニケーション・アーキテクチャの理解
    - Good First Issue⇒1画面まるごと作成へ
- 2か月目
  - 1日あたりの目標PR数の設定
  - オンボーディングの中間振り返り
    - チェックリスト形式で残タスクを可視化
- 3か月目
  - 毎日1on1⇒週次の1on1へ移行
  - 大きめの機能開発を一から担当（画面設計～実装まで）
- 文化
  - timesで呟けば誰かが助けてくれる
  - 週2回の10分勉強会（チームメンバーの持ち回り）
    - AIツールの使い方や情報交換が活発
  - 今後関わりそうな人（ビジネスサイド）に気軽に1on1を申込可能
- 3ヶ月後
  - フルリモートもAI活用もできた
