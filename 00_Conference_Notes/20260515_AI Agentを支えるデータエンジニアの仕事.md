# 「いつの間にかデータエンジニア以外の業務も増えていたけど、意外と経験が役に立ってる」
- データエンジニアになる前のキャリア
  - 新卒で数十人規模の小さなスタートアップに新卒入社
    - Rubyを書くバックエンドエンジニア
    - たまにインフラ仕事
  - その後の子会社に行って、何度か社名が変わって今
    - 2017年、M&A直後に突然会議室に呼び出され、「ZOZOのデータをBigQueryに集めてほしい」と言われた
      - 数年後に何をしていたかが分かった
      - 2018年当時は、名前がない職種で、やっと名前が付きだした頃Σ（・□・；）
        - 名前が無いから故の苦しみがあった
        - 調べ方間違えて、根本が違うから全部駄目になったりもした
- 当時は、バッチで入れていたから、1日1回だった
  - 要望により、高度な業務にも挑戦
- データエンジニア以外の仕事も
  - 自分が取りに行った訳でもないけれど、Google詳しいと社内に広がった結果グーグルCLoud側の接続ポイントを管理とか
  - グーグルクラウドのCCoEとか
- データパイプラインの構築
  - Claude Code⇒Cloud Run⇒BigQueryという
  - OTelの仕様はオープンなので、各社で構築済みの環境にいれるのがオススメ
- 利用者に何もさせないという思想
  - データ基盤管理時代に培った思想
  - データ基盤管理者の悩み
    - ○○テーブルを削除しようと思いますが、使っている人は手を挙げてください
      - 誰も反応しない。消すと、数日後に苦情を言う
      - 裏でいい感じにやる方法を探す癖が付いた
    - 中央集権的に管理できる機能に対する関心が高まる
- 仕組みを布教するときの心得
  - 分析になれ始めた人
    - 明らかに違う・どこから説明したら良いか悩むレベル
      - 解説したくなる気持ちを堪える
      - 知識を伝える事だけに比重を置きすぎてはいけない
      - AI開発ツール導入も同様
    - 本当の目的である効率化に繋がれば何でも良い
- AIエージェント時代のデータエンジニアに向けて
  - AIが1級側で、人が2級になるのでは？
  - 定型的なコードを書く仕事は減り、設計・監督者にシフト
  - 具体的なタスクをせずに抽象化された能力を伸ばす必要
    - 昔は、1・2か月かげて作業する事が許されていた
      - 特に新卒から2，3年目の人
- 個人的には、データエンジニアは、潰しが効く
  - 組織横断で仕事をするスキル
  - 全体最適化のスキル
