# 19:40	AI時代の決済を2軸で捉える — x402エージェント決済、トークンベース従量課金の実装	岡本 秀高
- 去年まで、Stripeに居た
- 年間150本ペースくらいで記事を書く人
- 決済はずっと「人間が、ブラウザの前に居る」前提で作られていた
  - 操作：16桁を画面に手で打つ前提
  - 検証：３DS・生体認証
  - 判断：ストア
- 上記をAIに頑張らせるくらいならば、別の決済手段を作ろう
  - 今、AI決済戦国時代
  - １：選ぶ
    - 人がエージェントに、どのECに委任するか？
  - ２：支払う
    - エージェントが、自分の予算から、APIに直接
  - ３：継続する
    - SaaSを契約する
      - 誰が払う？
        - エージェントが、開発者のStripeアカウント経由でサードパーティSaaSの月額/年額契約
- １：人が委任
  - ACP/SPT/Link Agent Wallet
  - ACP：商品を見つける（mapのエージェント版・見つけやすい）⇒Compose⇒Pay：これがSPT（Shared Payment Tokens）
    - SPT：安全に個人のStripeから、そのECサイトのアカウントにお金を持っていくためのトークン
      - 4つの制約
        - 金額上限
        - 期限
      - SPTを発行・受領が、日本ではまだ不可だが、その前段階をやるだけでもAIから見つけやすくなる
- ２；機械⇒機械
  - x402/MPP
    - x402：1行で言うと、「金持たない奴は、追い返せば良い」
      - 概念は、1990年代から予約されてあったが、方法が無かった
      - コンテンツ系は、これに備えた方が良い
      - テスト：簡単にできる
        - 事前に支払い用のウォレットを登録しとく
        - リクエストすると、レスポンスの支払いを要求する
        - 登録されたウォレット
        - 決済が通ったら、200で返してくる
    - MPP
      - x402を内包する、マルチメソッド版
      - クレジットカードとかも使用可能
- ３：機械⇒SaaS契約
  - Stripe Projects
    - 32プロバイダー・45サービスが繋がる
      - Renderとかvercelとかある。Supabaseも
    - 使用してみた
      - １：作成中のアプリを登録してみた
      - ２：Stripeのskillsがダウンロードされて、SaaSを探してくれる
      - ３：日本がSPT使えないから、無料プランだけが落ちて来る
        - envも落ちて来るが、本番のAPIキーだけは、あなたがやってくださいとなる
        - 契約したSaaSを使用するためのチュートリアルのようなSkillsが更に落ちて来る
        - そのままデプロイできる
    - skillsが落ちて来るから、Cursorに推奨を選ばせようとしている
    - どんどん.envに突っ込んで来るらしい（笑）
    - **解約すると、本当にアカウントを消してくる**
- 日本は、まだstablecoin決済を「受領」する側になること
- 今、x402で追い返すと、対応していないサイトは、「見れません」となるだけ
  - どっちが対応するのが早いかだけ
# 20:20	Stripe Connect × LINE API でカゴ落ち率を6ポイント改善、GMV を10%押し上げた話（仮）	hirata さん（@haruya_em / lea PM）
- GMVを3倍にした施策
- フリーランス6年目
- LINEを使ったマーケティング支援もしている
- レアの月額決済：Stripe
- 各店との決済：Stripe connect
- GMV：ユーザー売上の手数料
  - ユーザー数（ショップ数）x1店舗あたりの売上
- ここ3年で3倍くらい上がっている
- LINE登録者数⇒ショップアクセス⇒カート追加⇒注文⇒リピート
  - LINE登録⇒注文：6.28%平均
- 自力でアクセスログを取得して、BigQueryに集約する
- レアは、配信分析機能があるから、それにも必要だったので、そのためのデータも取得
- １：LINE登録⇒ショップアクセス
  - セグメント配信をもっと活用してもらう施策
    - 普通のショップ運営者が自分で配信対象を考えて使いこなすのは難しい
      - 最初は、なかなか利用されなかった
      - 配信者に対象をオススメして、残りワンクリックまでにした
      - 7.4%⇒23.1%のショップアクセス率に改善
- ２：カート追加⇒注文
  - LINEにカゴ落ち配信
    - UIに拘って
    - Flex Message
      - 商品画像
      - 購入ステップの可視化（進捗）
  - かご落ち率：37.2%⇒31.1％
  - カート追加⇒初回注文：63.8%⇒68.9%
- ３：定期購入
  - Stripeで簡単にできました
    - Checkout Session、自動で3Dセキュアをやっといてくれていた
  - 購入してから、1ヶ月以内がリピート購入が多い
    - リピーター向け配信
    - 累計購入額多い人も自動リストアップ
- レアの顧客は、小規模業者なので、ITリテラシーが高くなくて、簡単にやっといて需要が多い
- 現在進行中の施策
  - Stripe Connect x PayPay
- Q&A どうやって、指標とかログ決めた？
  - 元々、LINEマーケをやっていたので、自力でやっていた事を実施
