# 新谷修平	実演：HireRoo システム設計形式の解き方・企業の評価視点と採点プロセス
- 「システム全体を俯瞰して意思決定する力」が市場価値を左右する
  - 数百万人が使うサービスでどう負荷を分散するか
  - 5年先も使用できるか
- HireRooのシステムデザイン形式は、最近導入増えている
  - エンジニアの思考を表示させることを意識している
  - 左：問題文と要件
  - 左下：メモ帳書き出し（思考を残せる）
    - マークダウンでメモしている
    - 制約やピーク負荷、設計方針等を記載
    - トレードオフ：デメリットも記載しとく
    - 運用時の事も書く
    - 企業が見たいのは、なぜ、それにしたか
  - 真ん中：設計に使用するコンポーネント
    - 使用したいコンポーネントが無ければ、類似を置けば良い（テキストで補足可能）
    - 意図が伝われば良い
  - 右側：ドロップ＆ドロップでアーキテクチャー設計
    - ドロップで持って行って、ダブルクリックすると更に詳細に選択肢がある
  - オートセーブ機能あり
  - 右上の「提出」押すまでは、何度でも変更可能
- 評価画面
  - AIによる解析で回答判定
    - 問題の可用性・スケーラビリティが満たされているか
    - どのコンポーネントをどのように配置したかの正解判定がある
  - どの順番で作成したか、リプレイ画面で録画が見れる
- トレーニングをHPに用意しています
- 200社以上導入されている
