# Oikonさんご講演
- 公式がベストプラクティスを書いているからCommon workflowから入って、Best Practiceをやろう
- 今回、図式をベースに話す（資料見よう！）
- CLAUDE.md階層構造
  - 賞味期限が短い情報は、書かない
  - Managed Policy CLAUDE.md
    - 一番強い。上書き不可。組織全体のポリシー
  - 最近増えたAuto Memory
    - 賛否両論
- 操作を許可・拒否で制御する仕組み（setting.jsonで設定）
- Hooks
  - 動作を制御する機能
  - 確定的な動作をしたい時に使用
  - 例：かなり細かいのが増えている
    - チームメイトがアイドル状態になったら、依頼するとか
- CLAUDE.md VS Hooks
  - タイプスクリプトとかの絶対させたい事は、Hooksの方が良い
- Skills（Slash Commandsの延長）
- Subagents
  - 独立したコンテキストで専門タスクを実行するエージェント
    - メインエージェントは、サブエージェントが何をしているかは知らずに、サブの結果が返っていく
  - 使用している人としてない人が分かれる
  - 相性が良いのは、サーチタスク等の書き込みをしないタスクが良い
- MCPサーバー連携
  - 外部サービスと標準プロトコルで接続
  - そろそろ昔の手法になっている
  - ものによっては使える
- CLIツール連携
  - 外部とのやり取りをする際、コンテキストの効率が一番良い方法（By.公式)
  - 例
    - gh（GitHub CLI）：PR作成
  - MCPとよく比較される
  - CLIを使う事を意識した方が良い
- Plugins
  - チームに配布可能なバージョン管理できる拡張パッケージ
  - 公式の配布が最近頑張っている
- Agent Teams
  - 複数のClaude Codeインスタンスが共有タスクリストとメッセージングで協調する
    - 2025年6月くらいには、個人でやられていたのを、公式が2026年1月にリリースしたような感じ
- Subagents VS Agent Teams
  - 違い
    - Agent Teamsには、チームリーダーとサブの間に共有タスクリストという共通がある
    - Agent Teamsには、サブ同士のやり取りもある
  - Agent Teamsの方が出来る事が多いと考える
- 個人的に意識している事
  - 参照可能なコンテキストを用意
  - ツールを整備する
  - Claudeに検証手段を与える
    - テスト、スクショ、期待出力を提供
  - 細部よりも全体把握する
    - Claudeに何をしているか聞く（別セッション、/fork、/rewind）
    - ASCⅡなど図を書かせる
## Q&A
- ChatGPT5.4が出ましたが、牙城は崩れないと思いますか？
  - グルグルと新しい物が出て、1位が変わっているから、気にしていない
  - 1・2年見た限りでは、何処かが突き抜けそうにない
  - ある程度のタスクは、もうどのツールでも出来るようになっているので、自分が使いやすい物で良いと思う
    - 時々上手くいかなかった時に、他を使うくらいで良いと思う
- 仕様駆動開発を検討しているが、どうですか？
  - 用途によっては、使うべき
  - 仕様駆動も色々あるけれど、原理原則を把握しているかが重要で、何利用しても良いと思う
- マルチモデルの使い分けはどうしていますか？
  - 割とOpusばかり使用している
  - 個人的に使いやすいからClaude Codeばかり使用している。
    - Codexとかは、レビューとかレイヤー違う事を任せている。最近、実装でも使う事も。
      - ６：３：１くらいで使っている
- AIファーストで開発プロセス全体を行う前提での環境整備の意識
  - AIがアクセスしやすいように情報を置く
    - マークダウンが読みやすい
    - GitHub issueが取りやすいから、そこにタスクを置く
    - ファイル名も名前で内容が分かるようにしている
- 細部を見ない事に怖さがあります
  - プロジェクトとか物によると思います
  - 絶対に落としたら駄目なシステムは、辞めとこう
  - 公式を見てても、何かバグが出ても凄い速さで修正されるから問題ないという感じ
    - サービスによっては、このスタイルが当たり前になるかも？
- Claude.mdの長さ
  - 公式も500行以下推奨
  - 長すぎると、重要な2行とかも500行と100行の中では、重要度変化する
- お気に入り・推奨されているSkills教えて下さい
  - ralph loop（ラルフループ）：長時間実行させるタスクのループ
    - 上記を使用するために、タスクを切ってくれるralph-interviewを自分で作った？
  - skill-creator：公式
  - コードレビューのスキルは、誰でも使いやすい
- TDD
  - 実装途中で見つけたタスクを途中で指示やHooksを入れたりしている
