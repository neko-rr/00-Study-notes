# 『実務で役立つ ログの教科書』	増井 敏克
- 本書いている人：30冊以上表示させてる
- 世の中にあまり無い本を目指して書いたのが今回
- ログの役割
  - セキュリティ
    - 事後対応
    - 不正防止
    - 予兆検知
  - ビジネス
    - 出退勤の記録
    - アクセス解析
    - 問い合わせ対応
- 大事なのは普段からログを見る
  - 異常を検知するには、何が「通常」なのかを知っておく事が大事
  - 毎日ログを集計すると傾向が分かる
    - 時間軸での集計
    - グループでの集計
    - 環境での集計
- ログを可視化する
  - ただログの羅列を見るのは辛いので
  - 生成AIを用いて分析できる
    - そのまま渡して、説明してくれる
      - 但し、正確性は検証が必要
      - でも、もうかなり専門家に近い説明
- ログを日常的に見ていると、ITエンジニアならば、自然と問いを立てて効率化しだす

# 『7日間でハッキングをはじめる本』	野溝のみぞう
- 女性
- 趣味でセキュリティをやっている
  - 元々エンジニアではない。仕事はセキュリティ関連の事をコロコロと変えてやっている
  - 現在：8刷り：2万部
    - 最近、サービスの画面変わったから、差し替えた
- 大規模ランサムウェア攻撃が一般的に
- 初期侵入
  - 高度な攻撃というよりは、シンプルに弱い所から入ってきてるように見える
  - 攻撃側と防御側の非対称性
- 攻撃には、流れがある
  - サイバーキルチェーン
  - 著書もその流れを辿る
- ハッキングって？
  - Offensive Security：サイバーセキュリティの一分野
-  ハッキング学習には壁がある
   - 理由：日本では、法律に抵触する可能性がある
     - 初心者ほど環境を作るのが難しい
   - Try～：サービス利用して、安全に学習可能が売り
   - エンジニアであれば、理由は知っているが、「やった事ありますか？」
   - 周りに教えてもらわなくても新人でも学習できるものを目指した
# 『バックエンドエンジニアを目指す人のためのRust』	安東 一慈
- 不動産テック企業勤務
  - 不動産データを扱う社内システムをRustで作っています
- 実用的で楽しく身に着けるがコンセプト
  - メモリ付き電卓
  - 家計簿
  - 言語だけでなく、バックエンド全般の知識が身につく
- 発売から1年経過して
  - AIによりエンジニアを取り巻く状況が大きく変わった
  - 設計や判断の重要性が増している
  - 開発に必要な情報を全てをAIに渡せるわけではないので専門的なスキルや知識は依然として必要
- 本書を今読む理由
  - AIのアウトプットを確信を持って判断する第一歩
  - 堅牢なバックエンドを生産性高く作るための武器として
