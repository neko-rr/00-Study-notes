# Kaggle Grandmasterによる講演①画像検索モデルの開発ノウハウ	土井 賢治 / LINEヤフー
- 学習の基本：対比学習
- 商品画像は、正方形ばかりじゃない
  - 固定解像度入力だと「クロップ」or「余白」が発生しやすい
  - 入力例：細長い箒
- なぜSigLIP2
  - NaFlex
    - アスペクト比を保ったままViTへ入力
    - パッチを縦横に必要な分だけ並べる
    - ViTは、パッチ数（＝トークン数）に応じて計算量が増える
      - 社内の採用は、196
    - NaFLexの内部動作：Positional Embeddingのリサイズ
- 学習データ
  - full版（社内向け）
    - 社内の商品データから作成
  - OSS版（社外公開・OSS向け）
- 評価：社内評価データ＋公開ベンチ（ILIAS）
  - 指標：上位20件で正解が上位に固まるほど高い（最大が1）
  - 人手で収集したクエリ画像と正解画像
  - 複数カテゴリで平均
- まずはシンプルにファインチューニング
  - 解像度が大きい程、データが大きい程スコアが上がりやすい
  - この時点で、旧モデルよりも良くなっていた
- 課題モデル全体をfine-tuneすると性能が低下する場合がある
  - 提案
  - 2段階fine-tune
    - 部分unfreeze⇒低
    - どこを学習すると効く？
      - patch embeddingを学習（性能低下）
      - いくつかのパターンを全部比較
        - 社内評価データは、昔が0.6で、今回のモデルはどれも0.9付近であまり意味なくなっていた
- OSS版のモデルを公開しました
  - Apache2.0
  - ILIAS公式ベンチマークとの比較でも良いです
# Kaggle Grandmasterによる講演②「NFL Big Data Bowl 2026 - Prediction」金メダル解法	田中 良佑 / LINEヤフー
- 2025年新卒入社Σ（・□・；）
  - ショッピング部門の検索
- ソロ金：8位
- Kaggleアカウント：tk
- 選手のパス予測コンペ
  - 感想：私がバッジ取得のために参加だけして、スルーしたコンペ
  - 評価指標：一般的な時系列タスク
- コンペのポイント
  - 可変長の未来予測
  - 選手の時系列情報と選手間の相互作用をモデルにどう与えるか
  - 時系列系のNNが強いコンペ
- 解法
  - 時系列情報と選手間相互作用を分けて扱う構造
    - Temporal Encoderで各選手の時系列変化を捉える
    - ⇒Player Encoder
      - 選手方向にTransformer Encoder
    - 他の上位解法もTransformer Encoderを使用していた模様
    - 30フレームを入力し、40フレームを予測
      - 絶対座標予測ではなく、フレーム間の差分を予測
      - absolute = cumsum(delta)+last position
- 解法の工夫点（効いた事）
  - Player EncoderのAttention scoreのbiasを選手間のユークリッド距離ベースにすることで、自身に近い相手ほど相互作用しやすいという事前知識をモデルに与える
  - 前処理・データ増強
    - 攻撃方向を右向きに統一（x flip）
    - y軸反転augmentation & TTA
    - 時間シフトによるaugmentation
      - 入力系列の最後の数フレームを切り取り、予測ラベルの先頭に結合する事で、疑似ラベルを作成
      - 理由：長時間フレーム例が少なかったから
- その後は、ネタ切れで、モデルのバリエーションを無理矢理増やしてアンサンブルして金圏に到達
  - 最終日に10foldのモデルのfold0,1を追加
- 他の上位解法
  - 特徴量少なめ＆NNモデリングで工夫
    - 時系列方向
    - 選手方向
    - モデル構造
  - 時間シフトAugementation
    - 1st：パスの時刻をモデルに追加
- 生成AI活用
  - CodeXのみ使用
    - モデルは、GPT-5.1-Codex-Max
    - 精度改善案の質において、Claude COde(Opus)やGeminiよりも良かった
    - 解法の工夫点の「距離バイアス」を提案してくれた
- モデルの学習時間少な目で精度改善サイクルを高速で回す事ができるため、時間がない社会人にも優しいコンペ
- Q&A　特徴量の取捨選択方法
  - 最初、lightGBMで組んで、いくつか試して、効いたものをピックアップ
  - ピックアップしたものをニューラルネットワークに突っ込んだ
