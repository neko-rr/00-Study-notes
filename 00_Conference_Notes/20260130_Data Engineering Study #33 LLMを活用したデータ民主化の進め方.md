# 講演①「LLMアプリケーションの透明性について」
- 会社宣伝：一番大きいプロダクトは、「commune」
  - コミュニティプラットフォーム
- Community Sage
  - 分析エージェント
  - BigQuery Toolset　⇔Agent　問い合わせ回答
  - Google-adk⇒BigQuery
- Commune Pathfinder
  - クライアント調査アプリ
  - クライアント情報を各項目で並列調査
  - 調査結果をもとにして、コミュニティ提案のアイデアを出してくれるアプリ
  - メール文面の提案なども可能
  - 最低限の情報を自動で揃える
- Commune　Deal　Lens
  - 事例調査アプリ
  - 社内に蓄積された導入事例調査
  - ルールベース＋ベクトル検索
  - 調査内容を元に営業内容の提案も行ってくれる
- 透明性を意識
  - Cursorの考え方を参考に
  - 例：分析エージェントで、SQLが分からないユーザーにもある程度理解できるように説明させる
    - エージェントの挙動をオフライン評価して、モデルの性質等を分析
  - 例：クライアント調査アプリ
    - 調査結果はストリーミングで表示して、すぐに確認できるように
  - 例：事例調査アプリ
    - 各事例は事前にLLMで分類されたカテゴリが付与されている。各カテゴリツールの詳細が表示される
    - マッチング度を表示
    - どの事例を使用して提案したか、開示機能
    - 気に入らない場合は、指示を追加して再生成可能

# 講演②「データ民主化のための LLM 活用状況と課題紹介（IVRy の場合）」
- 電話の自動応答「アイブリー」
  - 累計5000万着電
- アイブリーを使用したデータプラットフォーム「IVRy Data Hub」
- データの民主化まだまだです
  - データ基盤にdatabricksを使用
  - 問い合わせ回答のAIあり
  - コンテキスト毎にAI用のスペースができるので、結果的に正しいデータへのアクセスになる
- どう作っているか：アーキテクト
- 社内分析用データのパイプライン：普通との事
  - 用途別にマートを区切って
  - セマンティックレイヤー：中間的な翻訳レイヤーとして機能⇒普段使っているようなビジネス用語でデータ利用可能：LLMと相性抜群
    - Unity Catalogメトリクスビュー
    - ダッシュボードで使われている範囲でAIが動作＝目的のコンテキストに沿ったデータにアクセス可能
    - データウェアハウスのアンチパターン？
      - サイロ型データマートは技術負債の一例ですが、試し中
      - 昔は、同じ言葉を使用していたのに、最近は少しずつ部署毎にずれている
  - できていないこと
    - セマンティックレイヤーはAIとの相性は良い⇒でも難しい
    - ビジネスの理解も必要で、まだまだ現場レベルのモデリングができていない
    - オープンでないデータの存在
      - スプレッドシートへエクスポートされ、その中でくみ上げられた分析データ
      - お試しのサンドボックス環境で業務で使用するデータが作り上げられた
      - LLMで何でもできるから、それっぽいのができるが、不安
    - アナリティクスエンジニアって、自分が何しているか分からなくなる人種
      - データの人と見られるが、課題は、人に関する事でデータ以外が多く何やっているんだろうとなる
- トップダウンで名指しで、ちゃんと見ろと言う会社なので、改革は進めやすい
- Q&A：経営と現場で求めるデータが違うと思いますが、どうなじませていますか？
  - 今、本当に現場の現場には浸透していない。多分、その時には分ける
  - 分けないと、デモのようなノリで使用できずに指定が必要になるから
# 講演③「データのゆりかごから墓場まで、エージェントはどう関わり、人間はどう振る舞うべきか」
- 大阪ガスの宣伝
  - 社内では社内「コンサルティング」集団と言っている（対外的にはデータサイエンティスト）
  - ちゃんと使用されて、評価されるで、運営
  - 今までは、個人商店状態で各人が各プロジェクトに体当たり
- データ基盤の在り方を考える
  - データのゆりかごから墓場まで
  - 意外と大事なのが「棚卸」：削除
  - TROCCOで全て解決：転送・加工・保存
    - GUIベースかつユーザーフレンドリー
    - データエンジニアの敷居を下げる
- AI駆動でデータパイプラインのライフサイクルも自動化する
- AIエージェントの活用で分析も自動化
- エラーも自動復旧
-Q&A：自動で復旧できるエラーとは、どのような物ですか？
  - 現状、半分くらい自動復旧できている
  - 運用初期で、設定不足が多い頃
  - 単に権限不足だったとか
- Q&A：新卒の教育どうしていますか？いきなり上司のようにAIに指示できますか？
  - 今までなら5年目あたりから教えてたマネジメント研修のAI版を行っている
