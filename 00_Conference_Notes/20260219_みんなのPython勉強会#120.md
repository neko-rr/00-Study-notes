# 『生成AIとPythonでラズパイカーを動かそう！ 〜LangChainとGradioで作る音声・画像操作のプロトタイプ〜』Tomoko Furuki（株式会社ビープラウド）
- **感想：聞いた事ある人かも？（女性）パイコンの人に似ている気がする。ランプとか指示が同じ。ルート画像が一緒だ！**
- Gradio
- LangChain
- Microdot
  - ラズパイPico上でも動く軽量web
- 何を作ったか
  - 声や画像による指示で動くおもちゃの車
  - Pico Car：市販のキット（乾電池だけで動かしたかった：リチウムイオン電池使いたくなかった）
    - Raspberry Pi：リチウムイオン電池
    - Raspberry Pi Pico：乾電池だけで動く
- Raspberry Pi Pico WH
- 理由
  - Picoカー単体だとMicroPythonを使用する必要がある＝一部ライブラリが使えない
- 動作
  - １：Webアプリから音声を入力
    - Gradioで作成
      - JSON形式のデータを表示
      - UIとコードを同時表示で、丁寧に説明
      - 待ち時間の工夫
        - 途中のステップ毎に表示を更新
          - 文字起こしの結果・生成AIが変換したコマンドを返す・全ての結果を返す
  - ２：音声を文字起こし＋生成AIで指示データに変換
    - API料金が嫌で、ローカルでfaster-whisperを使って文字起こしをする
    - テキスト⇒構造化された指示データ
      - LangChain
        - これくらいのレベルならば、直接Open AIでも良いが、モデルの差し替えが楽
        - 指示ではない言葉を言われた場合は、「その言葉に対する感情を表す動作・色」に変換するようプロンプトで指示
          - 特に、色等の指示を与えていないのに、悲しみで青色ランプとか幅広く反応する
          - 生成AIはCSSの色名だけを出力して、コードでRGBに変換
  - ３：コマンドを受信＋実行
    - Pico上のMicrodotサーバー
      - Flaskみたいにコードが書ける
      - コマンドを処理を順番に実行：for
      - できない事：距離が指定できない（モーターの速さをGradioで設定画面を作り調整できるようにした）
- 難しかったのは、キットの使い方
  - 走っている途中に車輪が抜けたり
- 別例：ルート画像を与えて、指示する
- 別のキット：micro:bitを使う場合の制約：Macとmicro:bitは有線だとシリアル通信で簡単にデータ変換できる
  - 初めて触って、1日で突貫作成したので、上手い方法があるかも？
# 『Spec Kitではじめる仕様書駆動開発(SDD)超入門』どりらん
- 仕様書駆動開発(SDD)
  - 仕様書が唯一の情報源
- SDDの欠点と注意点
  - 初期コスト
    - 仕様書作成に工数が必要
    - 小規模・試作には過剰
  - コンテキスト消費
    - 仕様・計画で結構トークン消費する
- Kiro
  - AWS
- cc-sdd
  - 国産ツール
- Spec Kit
  - 特定のエージェントに依存しない
  - 基本的に順番通りにやっていけばよい
  - 各ステップでテンプレートとプロンプトを自動生成
  - 成果物単位でコンテキストを切り替え可能
  - １：Constitution
    - プロジェクトの「憲法」
    - プロジェクトの不変の原則を定義する
  - ２：Specify
    - spec.mdができる
    - Clarifｙ
      - 最大5つの質問を自動生成
      - 見落としがちな仕様の矛盾も検出
      - 回答を仕様書に反映
  - ３：Plan
    - 技術計画を策定する
  - ４：Tasks
    - 計画を小さな実装タスクに分割する
      - 結構40個や60個できる
  - ５：Analyze
    - 仕様・計画・タスクの整合性をチェックする
      - 6カテゴリで矛盾をチェック
    - Cheklistコマンドで、特に気になる所を確認できる
      - かなり細かい
  - ６：
    - Task to issue：GitHubにissueを作る
      - そのままだと大量のissueが作られる
- Tips
  - 仕様作成前のブレインストーミング
    - Feature Development：Cloud codeの公式。
  - 仕様の分割
  - 仕様書の分量が多い
    - AIに質問しながら調整するのが良い
- このスライドもSpec kitで作成している
