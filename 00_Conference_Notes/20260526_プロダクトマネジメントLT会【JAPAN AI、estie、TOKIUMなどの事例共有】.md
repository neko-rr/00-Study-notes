# JAPAN AI株式会社（ジーニーグループ）久保田 善行	Skillsをフル活用したプロダクト開発どう作り、どう使われるものにするか
- AIで”作る”は速くなった、でもPdMの仕事は楽になったか？
  - 結局、戦略に使える時間は、30％以下
    - 「火消し」と「ミーティング」に消える
- 同じ40時間で、週に+18時間判断する時間に使えるようになった
- AIに「業務の型」と「必要な道具」を後から読み込ませる仕組み
  - skill多用
    - コードベースを読んで、仕様と開発チケット
    - 議事録から「要望」と「論点」を自動抽出する
    - 抽出した要望を、判断軸に沿って優先度決定する
    - 要件から、検証用モックをすぐ作る
- skillの良いところは、改善ループを回しやすい
- 作りたい物を、イメージして共有できる状態にする
- めんどくさいタスクを楽にして、判断の時間を増やす
# 株式会社TOKIUM　木下 博揮	「捨てる力」がプロダクトを強くする
- 請求書管理と経費申請のエージェントを担当している
- プロダクトに対して、様々な要望が届いていませんか(´;ω;｀)
  - しかし、リソースが限られている以上、絞る必要がある
- AIでアウトプットを増やすコストは、もう極端に下がった
  - 人間がやる差別化領域：絞る
- AIと収束を行うが最後の「捨てる」だけは、業務リアルを知る人間がやる
  - AI領収書偽造は、刑法該当（私文書偽造・詐欺罪）
  - それよりは、同じ領収書を弾く仕組みの方が喜ばれる
# 株式会社estie　高橋 奈々	作るものを尖らせるためのAI活用
- 元々、カスタマーサクセスから入って、エンジニアになった
  - ビジネスサイドから入りました
- 困ってませんか？
  - 言葉だけのヒアリングだと、感想が抽象的でニーズの解像度が上がらない
  - かといって、開発を止めて、モックアップを作るほどではない
- それを解決するのは、Claude Artifact
- この機能だけで、PdM1人で作成して、見てもらって、感想をもらい。
  - アイデアを尖らせる事ができるように
# 株式会社Unito　たむたむ (田村俊樹)	作らないために、作る〜社内オペレーションのリファクタリング〜
- 作れる時代、判断が増える
- アプリ
  - 家は、1ヶ月単位のサブスク
  - ホテルは、1日単位のサブスク
  - その間を提供
    - 外泊申請すると、アプリ側が使用して、清掃して返す
- 先にディスカバリーが必要
  - 「作らないために作る」
    - 捨てていい試作品を作る
  - 使われないものを沢山作ってしまう
- 4ポイント
  - 捨てる前提のマイクロサービス
  - 現場がバイブコード、PdMが磨く
    - オペレーション担当が最速で自然言語で叩く
    - 酷いものが出て来るが、やりたい事が分かるので、更にブラッシュアップ
- 具体例：ホテルシフト管理アプリの中間改善
