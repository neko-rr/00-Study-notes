# Escape from Excel方眼紙 〜マークダウンで繋ぐ、人とAIの架け橋〜	山口 拓也（日本経済新聞社）
- 日経の課金決済プラットフォーム
  - 多様なサービスを支える課金基盤
  - 歴史と法対応で、複雑な基盤ｎ
  - 2015年からメンテナンスされ続ける設計書等のドキュメントがExcel方眼紙
    - エンジニアと非エンジニアを繋ぐ
    - 自由な配置
      - インデントの自由
      - セルでインデント
      - スペースでインデント
    - 図形で表現
      - 罫線＋セル結合
      - 図形
  　- バージョン管理できない
      - 多様なファイル名
    - 図形などの表示が崩れる
      - Googleスプレッドシートにしたり
    - 生成AI時代
      - コード生成は、AIなのに、ドキュメントだけ手作業
      - RAGやMCP等ができない
- マークダウンなら運用が楽になる
  - LLMの性能向上
  - 周辺ツールの充実
    - Sub Agents
  - マークダウン化の価値
- 変換フロー
  - １：PDF変換
    - LibreOffice + pypdf
      - 図形の再現性が高い＆Dockerコンテナ使用可能
  - ２：テキスト抽出
    - 取り消し線など構造情報も含め文言の正確性を担保する
    - openpyxlでシート毎に処理
      - フォント情報から通常テキストと取り消し線テキストを分離
  - ３：生成AIでマークダウンに変換
    - 4種類のコンテキストを渡して変換
      - プロンプト
      - PDFファイル
      - テキスト
      - 取り消し線テキスト
  - ４：生成AIに検証させる
    - チェックスクリプトを実行
      - テキストの内容が含まれているか
      - 取り消し線の内容が含まれていないか
    - AI自身が判断・修正
- Claude Codeで更に効率化
  - Skills
    - 全体のオーケストレーターとして使用
    - 一連の流れを制御
  - Sub Agent
    - 各エージェントがシート毎に実際の作業
- プロンプト
  - Excel方眼紙の表現パターンごとに、マークダウンでの書き方の説明とサンプルを人間が作る
    - Excel方眼紙は自由だから、新たな表現見つける度に変換方法を考えている
  - 図形は、可能な限りマーメイドで書かせる
    - draw.ioでマーメイドで表現できないものは、手書き
- 金額
  - Opus4.6
    - 約＄14ドル
- 改善
  - レビューがGitHubベースになり簡単に
  - 全文検索・テキストベースの恩恵を受けるように
  - 今後は、RAGのナレッジにしたい
- 使用感
  - コードで読み取れることは、もう書かなくていい
    - 逆にAIが混乱する
  - SSOT（Single Source of Truth）の観点で整理する必要がある：ドキュメント
    - 感想：非エンジニアがコード読めないのは変わらないから、マークダウンは用意しといて、開発に読み込むかどうかのような気もする
      - それか、コードと最小限のマークダウンを用意して、非エンジニア用に瞬時にコード情報を翻訳するか
  - ハーネスエンジニアリングは続く

  - AIがOCR

- 月次決算を締め、試算表や経営レポートを提供
- 法人カードに関する管理業務も需要がある
  - 出張するから、トップアップ：利用上限上げる？

  - この頃は、結構人手も入っていた
  - 一部顧客への提供開始から始めた
- AI Coworking Platform
  - 自分達で開始して運用している
  - 複雑な業務もAIと人間がバトンリレー形式で効率化
    - スタートしやすく、効率化の見通しが立ちやすいプラットフォーム
    - 全行程を自動化できなくてもOK！

  - ユーザーの期待：エージェント１
  - 作業指示：エージェント２

- Heartbeat Modelで24/365Worker
  - 巡回・収集⇒検知・分析⇒提案・修正⇒教育
    - チャットログを常時モニタリング

# SaaSの操作主体は人間からAIへ 〜経理AIエージェントが目指す深い自動化〜	西平 基志（株式会社TOKIUM）
- Webエンジニア
- 社名はダジャレ：「時を生む」からTOKIUM
- 半年で9体のエージェントを開発
  - AI経費承認
    - 承認する側は、全社員を対応しなければならない
  - AI出張手配
    - ホテルや新幹線
      - 会社の規定通りに手配するのは、大変
        - アンケートでは、1時間かかる
- 前：人が操作するSaaS
  - 手作業でUI操作
  - 人が判断
  - 浅い自動化：定型作業の効率化
- 今：AIエージェントが操作
  - 深い自動化
- AIの性能が上がれば、自動化は実現するのか？
  - No
  - なぜ？
    - 判断を含む業務として、コードレビューを例にして考えてみる
      - コンテキスト抽出を阻む3つの壁
        - GitHub外のコンテキストが不可欠
        - 同じ業務でも、会社やチームごとに判断軸が異なる
          - 厳格or動けばOKorリリースサイクル毎に異なる
        - 暗黙知
      - コンテキスト抽出速度
        - 三重苦：広いx細かいx見えない
        - コンテキスト共有に積極的な企業はAI化が速い
- 開発者がコンテキスト抽出の最前線に立つ：TOKIUM
  - １：商談同席
  - ２：経理部研修
    - 毎月実施
      - 新卒研修として、長めの実施も
  - ３：お客さんの業務を実際にやってみる
    - お客様から頂いた本物データで承認作業を行う等
  - 月1回開催ユーザーミートアップ
