# コミュニティ農園における気象データの活用に向けて	YuMake株式会社　佐藤・松浦
- 「琵琶故知新」：滋賀県の美しい棚田の景観維持
  - 耕作地放棄拡大による景観荒廃
  - 少子高齢化
- 農業後継者が参加しやすいようにする
- 地域の農業が6次産業化にシフトするきっかけになるよう、情報を活用する一つの経験となる仕組み
- 電源が無い地域でも活用したい
  - 全ての機器を太陽電池電源で賄う・通信環境も4Gの通信を活用する形で環境を構築
  - 風の流れを「吹き流し」を設置する事で見える化してデータ取得
- IoTカメラ*2・日射量センサー.etc
- 2つのマイコン使用（PICマイコン）・基盤は、発注して作成
- 気温・湿度・日射量・CO2濃度・気圧・今日の天気・写真2枚（夜でも写る）
  - 15分に1回更新
- 農作業の獣害対応も次年度からしていきたい
# 拡大する衛星データビジネスと衛星データプラットフォーム「Tellus」の役割	株式会社Tellus　菅谷
- PR担当。過去、あらゆる会社（JAXAとか）の広報担当
- Tellus
  - 元々、経済産業省により、さくらインターネットの事業としてスタート
  - なぜ必要か
    - 宇宙情報産業の必要性
    - 2024年運用中の地球観測衛星数予測：900機以上
    - 上記データをいかに活用するか
    - 海外でも衛星プラットフォームが増えてきているが、国内初でスタート
    - アカウント登録者数4万人越え
- 自前で人工衛星を打ち上げたりはしてないが、情報提供して頂いている
- 無料利用のブラウザ環境やAPI・GUI等を整備
- クラウド上にパッケージ化した解析環境を提供：ストレージ・高性能CPU・QGIS
  - 1アカウント毎に定額利用可能
- パートナーと新しい価値を創造する
  - 例：環境省の立案で、バスキュールというデザイン会社と連携して可視化
  - 例：不動産の㈱ウィルと連携して、衛星データから緑地マップ作成して、不動産の価値説明力を向上
  - 例：猛暑予測
  - 例：棒再利用事例（東日本大震災：浸水被害・地震による地殻変動）
- 法人向けサービス（Drive：衛星データ特化のクラウド・SatHub：衛星による新規撮影の依頼・Pipelines：データ分析の共有強化）
  - 従来、それぞれの業者に都度依頼して、断られまくる作業があったのを、一括化
- AIモデルを作成するための検証データを提供
- Q&A：会社毎に強みは異なる？

# 防災速報アプリ 災害マップとデータ利活用について	LINEヤフー株式会社　冨田
- IOSアプリエンジニア（2022年新卒入社なのにマネジメントΣ（・□・；））
- 7000万人ダウンロード
- 災害マップ
  - ユーザー投稿の情報提供
  - 異常が増えていくと、通知も来る
    - 通知からも投稿画面にいける
  - 避難所や給水所など拠点情報
  - 交通規制エリア
- メリット
  - 細かい情報エリアが見れる：例：停電
  - 迫りくる災害の規模感・肌感が分かる
  - 情報の空白期間を無くす（自らの手で情報を埋めていく）
- なぜ作られたか。
  - 緊急時は、住民の方からの情報発信が重要
  - 行政だけでは、手が回らない
- 投稿データの利活用
  - ユーザーへの通知（異常感知通知が自動通知：投稿量と情報量を増やす仕組み）
  - ニュース記事化（自社の他サービスに活用＆公式情報にリンク）
  - 社内での分析
    - 今後の災害対応への対応を決める判断材料
    - 社内で可視化
- Q&A：間違ったデータへの対応は？
  - 基本は、社内の人の目が入っていて、削除対応
  - 後、「報告」機能があり、ユーザーからの報告で社内の人が見て判断
  - 発災したら、24時間シフトを組んで対応する
    - 実際に青森の地震の際は、深夜にほとんどの方がオンラインになっていた
- Q&A：アプリの年齢層は？
  - 高めに感じる
  - 情報の取り扱いが厳しく、エンジニアだと年齢が見れないが言葉遣いが年配かな？と感じる
# 特別講演 スマートシティ分野における取り組み〜まちまるごとマーケティングプロジェクト	株式会社デザイニウム　前田
- 福島県会津若松が拠点
- 地域密着の地域DXを掲げている
- 会津若松は、昔からスマートシティを標語している
  - 昔に大規模工場があったが、撤退したので、これからは工場よりソフトだ！となった
  - どんどん人口が減っていっている
- 何ができるか
  - 地域の魅力を増す
    - それに、データを活用
    - 例：道の駅が赤字（コロナ禍・30人くらいの町）⇒マーケティングしたいが情報無い（そもそも、どこから客が来ているか知らない）⇒ほぼ車で来るから、ナンバープレートを物体検出（YOLOv5）でナンバー認識：APIがあったのでそれを使用⇒Lookerで識別結果を蓄積（安く使用できるBIツール）
      - 多いと1日1,000台とか来る
      - 会津が3割・県内ナンバーが半分
      - 休日は、県外ナンバーが増える
      - 施策の効果測定にも使用できた
      - POSデータから、品目分析
      - 車台数とPOSデータとの関連グラフ
      - 冬は、雪のため車特定可能数が減る
      - 公式サイト・SNSの反応・AIカメラによる来場者数・ふるさと納税：属性分析
      - 事業の効果測定・デジタル地域通貨の効果
      - 上記を全部一箇所で可視化
- スタッフの人でも見れるようにしたダッシュボード
  - マーケティング担当「毎日・全員見ろ。見てたら、その内何か気付くから、それを実践しろ」
    - 上記を拡大して、道の駅に納入している業者にもダッシュボードを開放
    - 生成AIを組み込んで、自然言語でデータを取り出せるようにする予定
- 上記の事をやっていたら、奈良県下北山村から同じ事をしたいと声を掛けられた
  - 季節によって波が激しく、それの補填を行政がしないといけない時があり、不安定
  - 屋外アクティビティが多いので、気象データに手を出したい
- スマートシティ
  - デジタル住民票サービスを作成
  - 最近だとNFTで住民票を作成する自治体も出ているが、話者は興味が無かった
  - 県が基盤を作成しているので、そこと連携している
- Q&A：ECサイトのダッシュボードを作成しているが、データに興味が無い人にデータを見てもらうヒントを知りたい
  - 事務所の中にモニターを置いて、誰でも見れるようにして、ついで見させる
  - 発注担当とかは、データを見る事を業務の必須事項に組み込んで、当たり前にしていく
  - 納入している業者だったら、自分の売上に直結するので、自発的に戦略を見るために見てくれる
    - 1位の製品を生産したり、逆張りで異なる順位の製品を来年生産したりとかを聞く
- Q&A：道の駅に気象の影響を感じますか？
  - 雨の有無で、かなり顕著な反応の違いがある
