# 最新情報インプット
- 資料配布あり（コンパスのHPにある）
- Agent Skills
  - 実際の作業を一連の流れとして実行できるようになる
    - 例：メールを送る等
  - 配布されている既存のスキルを使うことも、自分でゼロから作ることも可能
  - Agent Skills の実体は、プロンプトを書いた .md ファイルと、必要に応じてスクリプトや参考資料をまとめた1 つのフォルダ
```md
.github/skills/reviewing-code/    ← スキル名のフォルダ
├── SKILL.md                       ← AI への指示書（Markdown）
├── scripts/                       ← 自動実行するスクリプト（任意）
└── references/                    ← 参考資料やテンプレート（任意）
```
  - 自動発火（呼び出し不要）
  - 段階的読み込み（コンテキスト節約）
    - 常時読み込まれるのは各スキルのタイトルと説明文だけ（約 100 トークン/スキル）
    - 実際に必要になったときだけ、詳細な指示を読み込みます。これにより、スキルを増やしても AI の性能が落ちません
  - オープン標準（どこでも使える）
- MCP との違い
  - **スキルは「能力」、MCP は「規格」**
  - この 2 つは対立するものではなく、組み合わせて使う
- どんなスキルが作れるのか
  - 最近、流行ったのは、確定申告。オープンソースが出来ている。
- 1. コミュニティリソース 4 選
  - ① github/awesome-copilot
    - [github/awesome-copilot/skills](github/awesome-copilot/skills)
    - GitHub コミュニティが作成した Agent Skills 集です。
    - そのまま .github/skills/ にコピーするだけで利用可能
    - コードレビュー、テスト生成、ドキュメント作成など幅広いジャンル
    - 自分のプロジェクトに合わせてカスタマイズ推奨
  - ② anthropics/skills
    - [anthropics/skills]()
    - Anthropic 公式のテンプレートとサンプル集です。
    - Agent Skills Standard の策定元が提供する「お手本」
    - SKILL.md の書き方や構造の参考に最適
    - resources/ や scripts/ の使い方の実例もあり
  - ③ openai/skills
    - [openai/skills]()
    - OpenAI 公式の Agent Skills サンプル集です。
    - OpenAI が提供するスキルのリファレンス実装
    - 他ベンダーのスキルと構造を比較する際の参考に
    - Agent Skills Standard に準拠したフォーマット
  - ④ Skillsmp.com（スキル マーケットプレイス）
    - skillsmp.com
    - コミュニティが作成したスキルを検索・共有できるマーケットプレイス型サイトです。
    - カテゴリやキーワードでスキルを検索できる
    - 多種多様なユースケースのスキルが集まっている
    - すぐにダウンロードして .github/skills/ に配置可能
- ⚠️ 注意: セキュリティに関する重要事項 Skillsmp.com はコミュニティ主導のマーケットプレイスであり、誰でもスキルを公開できます。公式のレビュープロセスがないため、安全でないスキルが含まれている可能性があります。利用する際は以下を必ず実施してください:
  - SKILL.md の内容を必ず読み、意図しない動作がないか確認する
  - **scripts/ 内のスクリプトを実行前に必ずレビューする（外部への通信、ファイル削除、機密情報の取得などがないか）**
  - resources/ 内のファイルも含めて全ファイルを確認してからプロジェクトに導入する
  - 不審な内容があった場合は利用しない
# パネルディスカッション（昨今のAI活用についてカジュアルに語る）
- １：AIを活用してどう変わりましたか？
  - 簡単なスクリプトやコードレビュー、書き方は、自然言語で出来るようになった
    - 凄く楽になった
    - 働き方が変わった
  - コードを説明するのにコンテキストの説明に苦労していたが、AI任せでコードを見て説明してくれる（AI任せが増えた）
- ２：情報のキャッチアップどうしていますか？
  - 最初はTwitterで拾って、深堀はZenとか。
    - 社内で人から聞いて、カスタマイズしていく
    - テック系をたまに発信しつつ、テック系アカウントをフォローしている
  - AIエージェントで、ニュースを集めてもらう
- 生成AIを開発で使う事は、社内で推奨されていますか？
  - バリバリ使うように推奨されている
- ジュニアエンジニアや経験の浅いエンジニア～中堅くらいのミドルくらいまで仕事やキャリアは、どう変わってくると思いますか？
  - ジュニアの方が、キャッチアップが早いと周りが言っている
  - シニアというか、開発経験がある人の方が、指示が上手い（自分たちもまだ、AI無し開発経験がある世代）
- エージェントツールがサイロ化しています。社内でコントロールされていますか？
  - 皆、創意工夫していて、周りに質問したり、交流のきっかけになるくらい自由
- ３：昨今のAgentic Developmentについて不安や期待などを教えて下さい
  - AIのみ開発経験しかないと、ハルシネーションに気付きにくいと考えている
    - AIにやらせるためのキャッチアップが必要だと感じている
    - 開発が好きで、AIに仕事を渡したくない人は、開発スピードが落ちていると感じる
    - AIに任せる任せないの線引きが、まだ曖昧で、AIに任せるからこその学習も存在すると思う
  - 自分が変更したコードだけれど、何書いているか分からないよね？という時が増えてきているのが少し不安
    - 自分達の頭の中にある理解を書いている訳では無いならば、人要らないのではという不安がある
    - 今のリポジトリの戦略とかが、AIによってどのように変化がするかが気になる
      - GitHubは、その全てを握っていると考えているので、期待しています！
        - Microsoft：他のコーディングツールに無い利点は、その通りです
          - 社内では、設計からデプロイまでAIで、しっかり管理しようという流れ
          - セキュリティ強化に力を入れている
            - AIのみの開発経験では、抜け勝ちなため
- AIの進化で今後のホワイトカラーの仕事、雇用がどう変わっていくか？
  - 昔から考えるとPCの成長と一緒で、「使えるから凄い」から「使う内容が重視される」ようになっていくだけ
