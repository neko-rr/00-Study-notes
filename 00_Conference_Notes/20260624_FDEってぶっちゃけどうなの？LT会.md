# AIエージェントでBPOを作り替える。それを実現するFDE/Claude Codeエンジニアの仕事
- FDEの定義
  - 顧客に入り込み、固有の課題を”動く成果”に変える役割
- BPO
  - スケールを担保しながら顧客に取り込む作業
- 例：帳票のデータ化を、AIエージェント＋人手で解く
  - 一発AIに投げるだけでは、難しい
  - 弊社のやり方
    - AIエージェントで柔軟にデータ化⇒最後に人手（ES)の確認で精度を上げ切る
      - ES：人手確認を効率的に回す仕組み（同じ入力を2名で行い。一緒ならクリア）
  - 帳票１：支店名で処理が変わる
  - 帳票２：手数料が得意先ごとに違う
  - 帳票３：書式が違う
  - 等
- 業務を理解し、harnessに落とし込み、ESと接続する
- harnessを改善すると、ミスが減り、原価（トークン）が下がり・スケールしやすくなる
- ラストワンマイルまでやり切って、成果を納品する：そこが価値
  - SWE x PdM x 業務コンサル x Ops
# FDEにしかできないことを無くしていく
- FDEって、受託や常駐なの？
  - 似て見える。でも、軸足はProduct側
    - 今後発生しうる課題も含めて、Productで解決する
    - 似た問題を抱える業界を全部横展開して、目の前の顧客を助ける
  - 感想：違いが分からない。。。
- 顧客課題を、プロダクトの進化に変える
  - 同じ道具で作る
    - ユーザーと同じように使う
  - 壁を見つける
  - プロダクトに戻す
    - 見つけた壁を機能や仕組みに変える
    - 汎用機能として戻す
      - 顧客専用とせずに、可能な限り汎用的な形にする
      - 一番難しい
        - 顧客に寄り添い過ぎると、一生FDEが必要になるから、バランス大事
- FDEがエンジニアの中で特殊なのが、唯一、ユーザーと直接話す
- FDEは、たくさん作る仕事ではない
- 1社の課題解決を、次のユーザーの標準体験に変えていく
