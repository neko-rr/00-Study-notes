# 「なぜ今、PDFはMarkdownよりJSONなのか」アドビ株式会社 デジタルメディア事業部 Document Cloud シニアソリューションコンサルタント：笹木 涼
- プリセールス
  - adobe前は、インフラのプリセールス
- AIはPDFをそのままでは、理解できない
  - 前処理が必要
  - PDF/PostScript
    - あくまで描画の情報で、内容が入っていない
- 前処理アプローチ（各社の対応）
  - １：ルールベース型
  - ２：構造解析型
  - ３：Vision Language Model型（最新）
- 問：Markdownで本当に十分か？
  - テキストと簡単な構造しか表現できない
  - PDFに含まれる豊富な情報が抜け落ちるのでは？
  - 例：サブタイトルとか、属性
    - 表の構造
    - 数値精度
    - 複合条件
    - KPI分析
    - 欠落検出
- 検証
  - 業務文書3種類（軽量PDF）＋政府文書（重量PDF)
  - 浅い15問＋深い15問＝30問
  - 4文書ｘ2形式ｘ3AIｘ30問＝720リクエストの全自動パイプライン
  - Cloud Codeでフルオートでやりました
- お試し検証結果：軽量PDF＆簡単な質問
  - Markdown：89.6%
  - JSON：43.1%
- 上記の結果は、おかしい
  - メタデータ・座標情報がノイズ
    - 本文10%以下
  - トークン爆増
  - 表データの深いネスト
- クリーンJSONを作った
  - 400kB⇒50kb
  - 結果
    - 簡単：89.3%
    - 難しい：47.7%
  - Markdown：の難しい：45.1%
- 重量PDFは、クリーンなJSONの方が有利
  - マークダウン（簡単）29.6%
  - 難しい：9.6%
  - クリーンなJSON（簡単）：48.7%
  - 難しい：35.5%
- 使い分けが大事

# ゲスト講演 「価値の再定義 — AI時代における「提案」と「共創」の新しいルール」Tably株式会社 代表取締役：及川 卓也
- あなたのKPIは、何を測っていないか？
- 価値
  - G-D Logic
    - 提供する
    - 作る人⇒使う人
    - 価値は、モノに宿る
  - S-D Logic
    - 作る人⇔使う人
    - 価値は、使う瞬間に生まれる
    - 20年以上前から提唱されている考え方
- 価値は、使われた瞬間に、文脈の中で生まれる
- 人⇔AI⇔人の共創エコシステム
  - ひとりでには起きない。

    - Slackの会話
    - 立ち話
  - ストック型
    - AIに渡せるナレッジは、ストック化されたものだけ
- ナレッジは増え続けるのに、組織の理解の総量は、むしろ減っていく
  - 人間の仕事
    - Core：何のために存在するか
    - Why：なぜ作るのか
- 創る
  - 「作る」：指示されたものを作るだけ
  - 目的地を定める
  - ガードレールになる
- 価値の総和を最大化する
  - プロダクト開発の基本フレーム
  - 功利主義
- 例：PayPalのボット集団
  - 社会・人類にとって、本当に良い物は何か？
- 「止めた人」を倫理的に判断できるかが重要
