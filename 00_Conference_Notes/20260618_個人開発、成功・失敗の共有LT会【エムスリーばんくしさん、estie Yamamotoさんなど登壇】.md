# 株式会社estie Ryosuke Yamamoto	12 年間で 1,500 万 DL、持続可能な趣味プロダクト
- 好きな物
  - 地理データ、Unicode
- Unicode Pad
  - この人が開発しているアプリ
  - 自分が欲しいを実装したアプリ
    - 無料：クライアントで完結apkさえあれば動く
    - アップデートは、年に数回
    - 累計１５４０万だうんろーど/MAU25万人
- 開発きっかけ
  - 内輪で流行っていた顔文字を簡単に開発したかった
  - 友達に簡単にUnicodeで変な記号を簡単に出せるものを配布したいからアプリ化
  - ついでに全世界におすそ分け
- 流行のきっかけ
  - 南米や東南アジアの人がSNSで装飾するために、キーボードがリッチじゃないから、これを使用する
- 継続には、モチベーションの維持が全て
  - 自分の欲しい機能を実装する
  - 高評価・口コミをもらう
  - 沢山ダウンロードしてもらう
  - 儲かる
- このアプリでは？
  - 2014（内輪）
    - モチベアップ
  - 2018（流行）
    - ユーザーの要望に振り回される（ない）
      - OSS化したから、勝手に開発してPRしてくれたら、入れるからというスタンス
        - 翻訳機能等
- OSS化という選択
  - 収益化との相性は良くない
  - OSSコミュニティは、そうそうできない
    - 開発は、これまで通り気ままなまま
  - モチベーションが尽きた後も誰かが引き継ぐ可能性がある
- 年間稼働は、2，3日
# 株式会社スカラコミュニケーションズ 株式会社ソーシャル・エックス 赤神青空	個人開発でiOSアプリを作ったらバズったのでバズの要因を紐解いてみる
- 本名Σ（・□・；）
- 新卒2年目
- 個人開発2種類
  - エンジニアが面白いと思ったものを技術盛り盛りで作る
  - 身近の人・自分のために作る
- 友達が、友人と遊ぶときに真ん中くらいで遊びたい
- 要件定義
  - ざっくり友達の要望
  - 自分：サーバー管理したくない・完全無料、何も管理しないでつくりたい
- Appleから提供
  - MapKit
- 3日くらいで、仕事終わりに作成したものがバズッた
  - 一目で分かるアプリだった
- なぜ、こうなった？
  - 友達が非エンジニア
    - 欲しいか、欲しくないかを都度都度質問して変えた
  - バズると、ユーザーの声が沢山出て来る
    - コンセプトを殺す声
      - 渋谷を出して
      - 複数候補出して
      - 検索機能を付けて
      - ⇒便利にすれば、Googleマップに勝てない
    - コンセプトに合う声
      - 知らない駅が出て良かった
- 「できる」と「すべき」は違う
  - できるから機能を付けても誰も使わない
  - 技術力があるほど、引き算の判断が難しくなる
  - 起点が設計を決める
# モチヤ株式会社 梅木 和弥	「厚いFW・ORM」の誘惑と技術選定のリアル
- 厚いFW・ORMって、最初は最高
- でも続けると。。。
- 厚い⇒薄くに乗り換えた
  - 事例１：Gin+Gorm⇒net/http + sqlc
  - 事例２：Next.js+ライブラリ盛り盛り⇒Lit＋薄いライブラリ
- FW/ORM/ライブラリは、”お手本コード”に過ぎない！！
- お手本（FW）は、誘惑
  - ウォッチコスト
  - 破壊的変更の認知コスト
  - お手本で済む？え、ものづくりする意味って。。。
- 破壊的変更による虚無業務でプライベートを費やしたくない
- 厚い構成をやめて効いた事
  - 言語レベルベースでの開発
    - 調査・実装・運用、どれをとっても情報が出回っている
  - ボイラープレートを育てる⇒資産になる
    - 複数やってたら基盤が似た構成になってきやすい。自分だけのテンプレートを育てられる
  - 言語レベルの習熟は、本業に直結
- お手本を借り続ける・読み続けるか、自分で土台を作るか
