# 「型定義でAIと会話する: 型を通じてAIに意図を伝えるテクニック」	株式会社カオナビ 鈴木 僚太 氏
- AI時代のプログラミングにおいて重要になりそうなこと
  - AIに的確に意図を伝える事
  - AIの書いたコードを素早く理解する事
    - 諸説あるが、少なくとも現在はそう
- AI開発のよくある課題
  - AIが出してきたものが自分の望み通りか素早く判断したい
- 型という救世主
  - AIが「型を申告」してくれれば素早く気付ける
  - 型の役割
    - 今では人間とAIの共通言語のひとつとなった
    - 自然言語よりも、意図を高効率で伝達できる
- AI⇒人間：成果物の素早い確認
- 人間⇒AI：効率的に要件を伝える
  - ガードレールとする
- 型は要件であると同時にコードの一部である。つまり、必要に応じて変えなければならない
  - ガードレールに意図を込める：変えて良い型と変えてはいけない型
    - 意図を明記する
    - 型レベルのテストが有効な場合も

# Q＆Aセッション・クロージング
- Q&A：「型の意図」をコメントに書くと、コードが冗長になり、逆にAIがノイズとして処理してしまう懸念はありませんか？
  - 基本は、命名規則で分かるのが一番は肯定です。
  - 一番効果的なのは、AIに誤解されそうな所だけ書くのが良い
    - 人間として考えても分かりにくいような所
- Q&A：Storybookを維持し続けるコストが大きい印象
  - 運用は考えてない
  - 従来だと、負債になって誰もメンテナンスしないとか聞くが、私はガーと一気に作る時に使用して、使わない段階に来たら捨てる
- Q&A：結局、見た目に影響出ている部分を確認するために、もう一度見る必要があるんじゃ？
  - 株式会社カオナビでは、逆にファイル単位でモックするなと指導していて、真逆。
  - Ubie株式会社では、フロントとバックでリポジトリが分離していて、モックしないと検証コストが大きい。
