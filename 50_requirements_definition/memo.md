# ペルソナ設計
- ユーザー像を明確にすることで、「誰のためのサイトか」がはっきりし、伝えるべき価値や必要な情報設計が明確になる
- ペルソナは、商品やサービスのターゲットとなる架空のユーザー像を具体的に設定する手法で「ある特定の個人」を設定することが特徴
- ターゲットが「20代大学生」「30代主婦」といった大まかな属性を示すのに対し、ペルソナでは年齢、職業、居住地、行動パターン、価値観などを細かく設定
## 主な設計項目
- 基本属性：名前、年齢、性別、家族構成、職業、年収、居住地など。
- 行動の傾向：趣味、一日の過ごし方、休日の行動など。
- 内面の傾向：性格、価値観、悩み、目標、課題の背景。
- 情報収集の傾向：使用デバイス、情報源、インターネット利用頻度など。
## ペルソナ設計のメリット
- ユーザー中心の設計が可能になり、使いやすいサイトやサービスを作れる。
- プロジェクトメンバー間で共有することで方向性のズレが減り、手戻りが減少する。
- ユーザーの課題を具体的に引き出しやすくなり、改善策を立てやすい。
よくある「ターゲット」との違いを見てみましょう。

|  | ターゲット | ペルソナ |
|---|------|-----------|
| 定義 | ユーザーの集団 | 架空の個人 |
| 表現 | 「20〜30代の健康意識が高い女性」 | 「田中 美咲、32歳、都内在住のWebデザイナー。最近リモートワークで運動不足を感じている...」 |
| 効果 | 属性はわかるが、行動や感情は曖昧 |  |

# カスタマージャーニー作成
具体的にペルソナを設計できたら、続いてカスタマージャーニーを作成
- カスタマージャーニーは、ユーザーの行動を可視化した表
- 商品やサービスを認知してから購入に至るまでの行動や心理の流れを時系列でマッピングしたもの

## カスタマージャーニー作成のポイント
- ゴールの明確化：ファッションECなら「服を購入する」、フィットネスなら「体験予約をする」など、ユーザーが達成すべきゴールを先に設定する。
- フェーズ分け：一般的には「認知 → 興味 → 比較 → 申込 → 利用後 → 継続」の6フェーズに分けて整理します。
- 感情を可視化: 行動だけでなく、「期待」「不安」「満足」など感情の変化も記録することで、UX改善のヒントが得られます。
- タッチポイント整理: SNS・検索・口コミ・メール・LINEなど、ユーザーと接する接点を明確にしておきます。
- AI活用の意義: AIに初稿を出してもらうことで、抜け漏れのない仮説を短時間で構築できます。その後、人の観察と洞察で磨き上げます。
## カスタマージャーニー作成：生成AI活用
```md
ペルソナに対してカスタマージャーニーを作成してください。

【ペルソナの概要】
"内容"
 【出力してほしい内容】
 認知 → 興味 → 比較 → 申込 → 利用後 → 継続 の6つのフェーズに分け、各フェーズで以下の項目を表形式で整理してください。 
・そのフェーズでの行動 
・抱く感情や思考
 ・利用するタッチポイント（SNS、検索、口コミなど）
 ・情報源 ・KPI（企業側が測定したい指標） 

分かりやすく1行ずつ表形式で出力してください。
```
# 要件定義の主な項目
- 目的・KPI：サイトやサービスの目的とそれを測る指標を設定。
- ターゲット：作成したペルソナや主要ユーザーグループを記載。
- 機能要件：フォームや決済機能、CMS、ログイン機能など必要機能を具体的に洗い出す。
- 導線設計：主要なページ間の遷移やCTAを整理し、ユーザーがゴールへ到達しやすくする。
- ページ構成：必要なページのリストと階層。
- コンテンツ計画：各ページに配置するテキスト、画像、FAQ、レビューなど。
- デザイン方針：トーン＆マナー、ブランド要素、配色やレイアウトの指針。

## ペルソナ自動生成ツール Insight Journey[https://insight-journey.web.app/](https://insight-journey.web.app/)
- 解決したい課題と課題を抱えるターゲットを入力すると、ペルソナ設計からユーザーのペインやゲイン、課題の背景、代替方法、ジョブ（本当に解決したいこと）、刺さるセールスワード、想定質問などを自動生成するツール
- Insight Journeyは、短時間で詳細なペルソナとその内面を理解するための仮説を提供する強力なツールですが、生成結果はあくまでAIによる仮説であり、現実の顧客データやインタビューを通じた検証が必要

## 要件定義セルフチェック
- 主要導線とページ遷移の流れ
  - ペルソナが目的を達成するまでの導線は明確か？
  - トップ→詳細→行動（申込・問い合わせ）の流れに無理がないか？
  - 主要CTA（ボタン・リンク）の位置や数は適切か？
- 必要なページ一覧
  - 必要なページをすべて洗い出せているか？
  - ページの役割（目的）は明確か？
  - 重複や不要なページはないか？
- 必要機能
  - フォーム・予約・検索など必要機能を把握しているか？
  - 各機能がどのページで使われるか整理できているか？
  - 実装や管理が現実的な範囲に収まっているか？
- コンテンツ案
  - 各ページで何を伝えるかが具体的か？
  - ペルソナが求める情報（信頼・比較・体験など）を満たしているか？
  - テキスト・画像・事例など素材の方向性が決まっているか？
- デザイン方針
  - サイトの印象（信頼・挑戦・親しみなど）は明確か？
  - カラー・フォント・トーンがターゲットに合っているか？
  - 他サイトとの差別化ポイントが説明できるか？
